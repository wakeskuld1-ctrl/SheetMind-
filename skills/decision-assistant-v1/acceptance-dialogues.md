# 决策助手 Skill V1 模拟对话验收稿

这个文档用于人工验收 `decision-assistant-v1`。

验收目标不是看 Skill 会不会“自由发挥”，而是看它是否稳定符合这 5 条：

1. 先解释风险，再解释动作
2. 已有 `table_ref` 时优先复用确认态
3. 不替用户直接开始建模
4. 只解释 Tool 返回，不自己下经营结论
5. 全程使用低 IT 用户能理解的话术

---

## 场景 1：用户只想知道下一步先做什么

### 用户

你直接告诉我，这张表现在最该先做什么。

### 期望 Skill 回复

当前判断：我先帮你判断这张表现在是更适合先清洗、先分析，还是已经可以进入下一步建模。  
下一步动作：我会先调用决策助手，给你一份按优先级排序的建议。  
对应请求：

```json
{
  "tool": "decision_assistant",
  "args": {
    "path": "tests/fixtures/analyze-distribution.xlsx",
    "sheet": "Metrics",
    "top_k": 5
  }
}
```

### 通过判定

- 默认走 `decision_assistant`
- 没有直接切回归或聚类

---

## 场景 2：上游已经给了 `table_ref`

### 用户

这张表前面已经确认过了，你别重复问我路径，直接给我下一步建议。

### 期望 Skill 回复

当前判断：这张表已经确认过结构，所以我会直接复用上游给出的确认态。  
下一步动作：我直接用 `table_ref` 进入决策助手，不再让你重复确认表头。  
对应请求：

```json
{
  "tool": "decision_assistant",
  "args": {
    "table_ref": "table_1234567890",
    "top_k": 5
  }
}
```

### 通过判定

- 发出的请求使用 `table_ref`
- 没有重新追问 `path + sheet`

---

## 场景 3：Tool 返回存在阻塞风险

### 用户

如果有问题，你直接告诉我最先处理哪一项。

### 期望 Skill 回复

当前判断：这张表里有几项会直接影响后续统计或建模的阻塞问题。  
优先动作：我会先告诉你最先要处理的那一项，再告诉你第二优先级动作。  
主要提醒：这一步先不要急着做模型，先把阻塞问题处理掉。  
建议下一步：如果你愿意，我再带你继续看具体是哪些列有问题。

### 通过判定

- 先解释 `blocking_risks`
- 明确说“先不要急着建模”

---

## 场景 4：Tool 返回可以继续建模

### 用户

如果这张表已经差不多了，你就告诉我下一步适合做什么。

### 期望 Skill 回复

当前判断：当前没有识别出明显的阻塞项，可以进入下一步分析。  
下一步动作：我会优先告诉你现在更适合继续看统计摘要、做聚类，还是进入回归分析。  
主要提醒：这是建议顺序，不代表我已经替你开始建模。  
建议下一步 Tool：优先解释 `next_tool_suggestions` 里的 ready 项。

### 通过判定

- 解释 ready 的下一步 Tool
- 不直接开跑模型

---

## 场景 5：用户开始问模型参数

### 用户

那你直接帮我做逻辑回归吧。

### 期望 Skill 回复

当前判断：你现在已经不是在问“下一步做什么”，而是要真正进入建模。  
下一步动作：我会切回分析建模流程，先帮你确认目标列、正类和特征列，再进入逻辑回归。  

### 通过判定

- 不继续停留在决策助手
- 明确切回分析建模主流程
