## 2026-04-12 Regression Head Precision Scheme C
- 当前回归头 readiness 偏松：
  - 只要 `minimum_sample_status == sample_ready`
  - 且 `valid_rmse / test_rmse` 存在
  - 就可能进入 `shadow_candidate_ready`
- 当前回归预测逻辑偏轻：
  - 每个特征命中 bin 后直接取 `predicted_value`
  - 最终把所有命中值做简单平均
  - 对薄 support bin 的极端均值缺少收缩
- 方案 C 的最小闭环可以落在两处：
  - 回归 bin `predicted_value` 向全局 baseline 收缩
  - readiness 增加“优于基线 + 方向命中达标”的治理条件
- 本轮已落地的回归增强是：
  - 回归 split 新增 `baseline_mae / baseline_rmse / rmse_improvement_vs_baseline`
  - readiness 新增 `regression_quality_status`
  - 回归 bin 预测值对薄 support 做 baseline shrinkage
- 当前 regression quality gate 采用的最小治理口径是：
  - `valid/test rmse_improvement_vs_baseline > 0`
  - `valid/test directional_hit_rate >= 0.55`
