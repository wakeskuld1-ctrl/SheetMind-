# 运维容量评估能力汇报版

## 这次交付的东西是什么

这次交付的是一套面向运维容量评估的 Excel 交付能力。

它的目标不是只做底层分析，而是让用户在不同数据条件下，都能较快拿到一份可直接用于评审和沟通的容量评估 Excel。

## 它主要解决什么问题

它主要解决 3 类常见问题：

- 数据零散，只有部分 Excel 指标，但又必须先判断是否有容量风险。
- 没有完整监控和部署清单，无法按传统方式做完整容量建模。
- 时间紧，评审会马上要开，需要先给结论、风险和下一步动作，而不是等所有数据补齐。

## 它是怎么解决的

这套能力不是只做简单线性估算，而是按不同证据条件组合使用几种手段：

- 有 Excel 指标时，直接读取业务量、实例数、CPU、内存等数据做容量判断。
- 数据不全时，结合服务等级、峰值模式、增长预期、冗余要求做弹性评估。
- 没有完整部署信息时，可通过受控 SSH 只读取证补充主机和实例事实。
- 证据仍然不足时，不强行给伪精确结论，而是输出补数优先级和决策路径。

## 它适合哪些场景

- 活动前扩容评审
- 上线前容量预审
- 压测前资源准备
- 监控尚未完善时的保守判断
- 需要快速形成运维决策意见的场景

## 用户最终拿到什么

用户最终拿到的是一份 Excel 报表，而不是一堆 JSON 结果。

报表固定包含 4 个部分：

- `结论页`
  直接看当前容量状态、建议实例数、瓶颈资源和推荐下一步。
- `资源测算页`
  说明 CPU、内存等资源维度的判断依据。
- `证据与风险页`
  说明本次结论基于哪些证据、当前证据强弱如何、风险在哪里。
- `补数与行动页`
  说明缺什么数据、先补什么、下一步怎么继续决策。

## 这次能力的核心价值

- 不要求用户一定先把数据补全，先有结论，再逐步补强。
- Excel 是主交付物，更符合运维评审和对外沟通习惯。
- 既能做量化判断，也能在数据不足时给出可执行建议。
- 通过受控 SSH 取证，能在缺少监控平台数据时补充基础事实。

## 当前边界

- 这不是完整监控平台替代品，长期容量规划仍建议配合更完整的历史指标体系。
- SSH 只允许白名单只读命令，不支持任意远程操作。
- 当输入信息很少时，输出会偏指导型，重点是决策路径，不是精确算值。

## 一句话介绍口径

“这套能力可以把零散的 Excel 指标、服务画像和受控 SSH 取证整合起来，输出一份可直接交付的容量评估 Excel。即使数据不完整，也能先给风险判断、补数优先级和下一步行动建议。”
