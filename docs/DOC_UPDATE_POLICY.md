# 文档更新规则

## 1. 适用范围

本规则适用于仓库内所有正式开发改动，尤其是：

- 新增或删除 Tool
- 修改 Tool 合同字段
- 修改主流程入口
- 修改 Skill 门禁
- 修改默认行为
- 修复会影响用户使用口径的 Bug

## 2. 默认必须同步检查的文档

每次任务完成后，默认至少检查以下文件是否需要同步更新：

- [README.md](/E:/TradingAgents/TradingAgents/README.md)
- [AI_HANDOFF.md](/E:/TradingAgents/TradingAgents/docs/AI_HANDOFF.md)
- [CHANGELOG_TASK.MD](/E:/TradingAgents/TradingAgents/CHANGELOG_TASK.MD)

## 3. 什么情况下必须更新 README

- 外部主入口变化
- 当前主链变化
- 默认行为变化
- 常用命令或推荐使用方式变化

## 4. 什么情况下必须更新 AI 交接手册

- 当前主线切换
- 新增正式 Tool / Skill
- 禁区或维护原则变化
- 近期已确认状态、风险、接手顺序变化

## 5. 什么情况下必须更新任务日志

- 任何代码改动完成后
- 任何 Bug 修复完成后
- 任何文档体系重构完成后
- 用户纠正了流程、口径、默认行为后

## 6. 执行顺序

1. 先完成测试与实现。
2. 再更新 README / AI_HANDOFF / 其他相关文档。
3. 最后把本次任务沉淀到 `CHANGELOG_TASK.MD`。

## 7. 额外要求

- 如果用户纠正了做法，要在任务日志中补“记忆点”。
- 如果是正式流程变更，不要只改局部说明，必须同步更新权威入口文档。
- 旧文档如果不再是权威入口，要改成跳转说明，避免继续分叉。
