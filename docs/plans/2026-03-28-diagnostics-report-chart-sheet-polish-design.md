# Diagnostics Report Chart Sheet Polish Design

## 背景

当前 `diagnostics_report_excel_report` 已经完成了第一版图表页增强：

- 默认生成 `图表摘要`
- 图表页内已落地真实 chart XML
- 已支持相关性图、异常占比图、趋势图
- 图表页可通过 `include_chart_sheet = false` 显式关闭

这条主线已经证明“Rust / exe -> workbook chart”完全打通，但图表页仍有两个明显缺口：

1. 左侧数据源区仍偏粗糙，字段够用但还不够像正式交付页。
2. 图表价值还不够满，缺少最有代表性的 `分布图` 和更直观的 `异常 Top 图`。

用户本轮已批准按方案 A 继续，只做图表页打磨，不回头重构。

## 目标

在不改变 `diagnostics_report_excel_report` 主入口的前提下，继续增强 `图表摘要` 页：

1. 把图表数据源区整理得更紧凑、更可读。
2. 新增 `分布图`。
3. 新增 `异常 Top 图`。
4. 保持现有图表页可降级、可关闭、可导出 `.xlsx`。

## 非目标

- 不新增新 Tool。
- 不修改底层统计诊断算法。
- 不改 dispatcher / catalog / workbook draft 主链。
- 不引入隐藏 sheet、多工作表图表编排器或 Python 运行时。

## 方案对比

### 方案 A1：只补分布图

优点：

- 改动最小
- 与现有 `distribution_section.bins` 结构天然契合

缺点：

- 图表页观感提升有限
- 异常图层次仍偏单薄

### 方案 A2：只补异常 Top 图

优点：

- 对“哪里最异常”表达直接
- 和现有异常占比图互补

缺点：

- 没有补齐分布页缺口

### 方案 A3：双补图 + 数据区整理

做法：

1. 保留现有相关性图 / 异常占比图 / 趋势图
2. 新增 `分布图`
3. 新增 `异常 Top 图`
4. 顺手把图表数据源列整理得更紧凑

优点：

- 仍然是交付层增强，不会碰主架构
- 图表页信息密度明显提升
- 对现有测试和业务合同影响可控

缺点：

- 图表数量增加后，需要更注意布局锚点

本轮采用方案 A3。

## 设计原则

### 1. 继续单 Tool 承载

所有增强继续留在 `diagnostics_report_excel_report` 内部，不新增额外 Tool，不分叉业务入口。

### 2. 图表只消费既有诊断结果

所有新增图表都必须直接来自 `diagnostics_result`：

- `distribution_section.bins`
- `outlier_section.outlier_summaries`

不允许重新跑算法或直接回读原始工作簿字段。

### 3. 图表页仍兼作数据源页

当前图表导出主线要求 chart spec 引用正式 worksheet。为了保持最小改动，本轮继续让 `图表摘要` 既是图表承载页，也是数据源页。

### 4. 数据区更紧凑，但不做隐藏列

本轮只整理字段结构和列顺序，不引入隐藏列/隐藏行逻辑，以免扩大导出层改动范围。

## 图表数据区设计

当前图表页数据区建议整理为以下结构：

- `相关性标签`
- `相关性系数`
- `异常列`
- `异常占比`
- `异常数`
- `分布区间`
- `分布计数`
- `分布占比`
- `趋势时间`
- `趋势值`

这样比当前版本更完整，原因：

- `异常占比` 与 `异常数` 同时保留，便于做两类图
- `分布区间 + 分布计数` 可直接支撑直方图/柱状图
- 左侧数据区更像一个清晰的图表源表，而不是临时拼接列

## 新增图表设计

### 1. 分布图

数据来源：

- `distribution_section.bins`

图表类型：

- 柱状图

X 轴：

- `分布区间`

Y 轴：

- `分布计数`

意义：

- 让业务方直接看到数据主要堆在哪个区间
- 补齐当前图表页对分布形态的可视化缺口

### 2. 异常 Top 图

数据来源：

- `outlier_section.outlier_summaries`

图表类型：

- 柱状图

X 轴：

- `异常列`

Y 轴：

- `异常数`

意义：

- 比异常占比更直观地告诉业务方“哪一列异常点最多”
- 与现有异常占比图形成互补

## 图表布局

建议保持两列图表布局：

- 左列：相关性图、异常占比图、趋势图
- 右列：分布图、异常 Top 图

锚点继续用固定 row/col 实现，不在本轮引入自适应排版器。

## 降级规则

### 合法降级

- 没有 `distribution_section`：不生成分布图，但保留图表页
- 没有 `outlier_section`：不生成异常相关图，但保留图表页
- 没有 `trend_section`：不生成趋势图，但保留图表页

### 整体不失败

只要 `diagnostics_report` 仍然能产出结果，`图表摘要` 就继续生成。

## 测试策略

本轮至少补 3 类测试：

1. `.xlsx` 导出后至少存在更多 chart XML，例如新增 `chart4.xml / chart5.xml`
2. `sharedStrings.xml` 中存在新的图表源列，如 `分布区间 / 分布计数 / 异常数`
3. 降级场景下没有 `trend` 时，图表页仍然导出

必要时再补：

4. `include_chart_sheet = false` 时仍不生成图表页
