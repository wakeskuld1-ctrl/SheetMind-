# Repo And Branch Governance

## 目标

这份文档用于说明当前仓库的对外展示方式、目录职责和分支治理规则，避免 GitHub 首页、分支列表和真实开发方向继续错位。

## 当前对外身份

当前仓库对外应以 `SheetMind / Excel_Skill` 为准。

历史来源、旧标题或旧演示内容可以保留为阶段痕迹，但不应继续占据 GitHub 首页叙事。

## 目录职责

### 核心目录

- `src/`: Rust 主代码与能力实现
- `tests/`: Rust 测试、夹具、CLI 验证
- `docs/`: 计划、验收、交接、执行记录
- `skills/`: 项目内 Skill 资产

### 历史或非当前主线目录

- `cli/`: 历史 CLI 相关内容
- `tradingagents/`: 历史来源内容，当前不是产品主线

保留这些目录不代表当前主线仍以它们为核心。

## 当前活跃开发主线

### Rust / EXE 主线

当前产品默认主线是 Rust / EXE / CLI-first。

### Foundation 主线

当前 foundation 导航内核已经推进到 retrieval 阶段，顺序固定为：

`ontology-lite -> roaming -> retrieval -> evidence assembly`

## 分支命名规则

建议继续使用：

- `main`: 稳定主分支
- `codex/<topic>`: 当前开发分支

`<topic>` 应该体现明确业务或工程含义，避免继续使用难以理解、已失效或没有承接关系的名字。

## 分支清理规则

### 本地分支

可以清理的情况：

- upstream 已失效
- 已经被其他活跃分支承接
- 已经合入 `main`
- 仅为临时验证或旧试验分支

### 远端分支

本轮默认不做激进清理。

只有在确认：

- 没有继续承接价值
- 没有未交付工作
- 名称已经失去业务含义

时，才进入下一轮远端整理。

## GitHub 首页治理

GitHub 首页应优先回答 4 个问题：

1. 这个仓库现在是什么
2. 当前主线是什么
3. 新人先看哪里
4. 什么是历史内容，什么是当前方向

如果 README 无法回答这 4 个问题，就应继续整理。
